Cloud Save

Manual restore points, not live sync.

Cloud Save is optional encrypted backup and restore. Your current device remains the source of truth unless you explicitly save or restore.

What Cloud Save does

  • Stores encrypted restore points when you choose to save.
  • Lets an existing connected device create a one-time code for a new device.
  • Lets a linked device restore the latest Cloud Save package using your recovery key.
  • Supports the default All Of Me Cloud server or a compatible self-hosted server.

What Cloud Save does not do

  • It is not live sync.
  • It is not collaboration or sharing.
  • It does not make the server the primary copy of your data.
  • It does not send the recovery key to All Of Me Cloud.
  • It is not designed so support can decrypt your restore points.

What gets uploaded

When you save, the app creates an encrypted backup package on the device and uploads that package with operational metadata such as save time, app version, payload size, checksum, and random account and device identifiers.

The encrypted package can contain the app data included in your backup, such as members, groups, notes, fronting history, and profile images. Treat Cloud Save access credentials and recovery keys as sensitive.

Recovery key

The recovery key is required to decrypt a Cloud Save package during restore. Store it somewhere safe. If the recovery key is lost, All Of Me Cloud is not designed to recover or decrypt the backup for you.
